## Overview

CiviClick is a web application that allows residents to report civic issues — potholes, graffiti, broken streetlights — simply by taking a photo. The app uses a fine-tuned YOLOv8 model to classify the issue type automatically and routes the complaint to the correct city department based on GPS coordinates.

## Problem

Reporting civic problems is tedious. Most cities have phone hotlines or complex web forms that go unused. Issues remain unaddressed because the barrier to reporting is too high. CiviClick reduces that friction to a single photo.

## Technical Approach

### Computer Vision Pipeline

- Fine-tuned **YOLOv8** on a custom dataset of 4,800 labeled civic issue images across 12 categories (potholes, graffiti, illegal dumping, broken signage, etc.)
- Achieved **91.3% mAP@0.5** on held-out test set
- Model served via **FastAPI** containerized in Docker for sub-100ms inference

### Geolocation & Routing

- Browser Geolocation API captures GPS coordinates on photo capture
- Coordinates reverse-geocoded against municipal boundary shapefiles to determine jurisdiction
- Routing table maps issue categories × jurisdictions → department email/ticketing endpoints

### Frontend

- React + TypeScript SPA with camera API integration
- Real-time classification feedback overlaid on image
- Submission confirmation with estimated response time

## Results

- Deployed at a Stanford hackathon; 120+ reports filed in 48 hours
- Demonstrated to 2 Bay Area city council offices
- Won **Best Civic Tech** award at TreeHacks 2025
