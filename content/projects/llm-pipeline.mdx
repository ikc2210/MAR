## Overview

A production-quality distributed fine-tuning pipeline for large language models (7B–70B parameters). Implements LoRA (Low-Rank Adaptation) for parameter-efficient fine-tuning, with multi-node training orchestrated via PyTorch Distributed Data Parallel and automatic fault-tolerant checkpointing.

## Motivation

Existing fine-tuning tools like HuggingFace `Trainer` and `trl` are excellent for single-GPU workflows but lack robust multi-node orchestration, automatic checkpoint recovery, and experiment tracking integration. This pipeline was built to fill that gap for research cluster environments.

## Architecture

### Training

- **LoRA adapters** injected into attention layers (rank 16, alpha 32)
- **QLoRA**: 4-bit NF4 quantization of base model weights via `bitsandbytes`
- **Gradient checkpointing** to fit 70B models on consumer A100s
- Distributed via `torch.distributed` with NCCL backend across nodes

### Data Pipeline

- Dataset preprocessing in `datasets` with multi-process tokenization
- Packing: variable-length sequences packed into fixed-length blocks for efficiency
- Streaming from S3 — training dataset never fully materialized locally

### Fault Tolerance

- Checkpoint saved every N steps to both local NVMe and S3
- Training resumes automatically from latest checkpoint on node failure
- W&B integration for experiment tracking and loss curves

### Evaluation

- Periodic evaluation on validation split with configurable metrics
- Supports MMLU, HumanEval, and custom benchmark integration

## Results

- Successfully fine-tuned Llama 3 8B and 70B on a domain-specific dataset
- LoRA reduced trainable parameters by **99.6%** vs full fine-tuning
- Achieved comparable benchmark performance to full fine-tune at 10x lower compute cost
